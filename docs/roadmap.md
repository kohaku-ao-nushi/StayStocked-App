## 技術的負債（優先度順）

| 項目 | 優先度 | 理由 |
|---|---|---|
| innerHTML XSS 対策 | 高 | セキュリティリスク |
| ユニットテスト導入 | 中 | calc 関数のバグ検知 |
| ストレージキーの移行（StayStockedApp → CrasonaApp） | 低 | マイグレーション設計が必要 |
| IndexedDB への移行 | 低 | localStorage の5MB制限対策 |
