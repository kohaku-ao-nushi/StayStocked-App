# 要件定義書 — クラソナ（Crasona）

> **最終更新**: 2026-05-31

## 1. プロダクト概要

**クラソナ（Crasona）** は家庭の防災備蓄を管理するモバイルファーストの Web アプリケーション。
「暮らしで備える。」をコンセプトに、家族構成・ペット情報をもとに備蓄品の推奨量を自動計算し、日常の中で無理なく備蓄を習慣化できる体験を提供する。

- **ブランド**: クラソナ / Crasona（暮らし＋備える＋ソナーのダブルミーニング）
- **対象ユーザー**: 防災備蓄に関心のある一般家庭
- **動作環境**: スマートフォンのモバイルブラウザ（PWA 対応済み）
- **データ保管**: 全データをブラウザの `localStorage` に保存。外部サーバーには一切送信しない
- **公開 URL**: https://stay-stocked-app.vercel.app

## 2. 機能要件

### 2.1 オンボーディング（FR-00）

| ID | 要件 |
|---|---|
| FR-00-1 | 初回起動時（`onboarding.completed === false`）はオンボーディング画面を表示する |
| FR-00-2 | 3ステップのスライドカード形式でアプリの価値を説明する |
| FR-00-3 | 「スキップ」または最終ステップの「はじめる」でくらし方設定へ遷移する |
| FR-00-4 | 完了後は `onboarding.completed = true` を保存し、次回起動時は表示しない |

### 2.2 くらし方設定（FR-01）

| ID | 要件 |
|---|---|
| FR-01-1 | 家族人数（1〜9人）を −/＋ ボタンで選択できる |
| FR-01-2 | 人数分の家族構成カードを動的に表示し、性別・年代を入力できる |
| FR-01-3 | 年代区分は「乳幼児（0〜2歳）」「子ども（3〜17歳）」「成人（18〜64歳）」「高齢者（65歳以上）」の4区分 |
| FR-01-4 | ペットを種類・単位・頭数で複数エントリ登録できる |
| FR-01-5 | 保存時に `localStorage` へ書き込み、ホーム画面へ遷移する |
| FR-01-6 | 保存済みデータがある場合は設定値をプリセット表示する |

### 2.3 備蓄リスト（FR-02）

| ID | 要件 |
|---|---|
| FR-02-1 | 備蓄期間モード（3日分 / 1週間 / 2週間）を切り替えられる |
| FR-02-2 | 家族構成に応じた必要量を品目ごとに自動計算して表示する |
| FR-02-3 | 現在の在庫量と必要量を比較し、プログレスバーで達成率を表示する |
| FR-02-4 | 品目をカテゴリ別にグルーピングして `<details>` アコーディオン表示する |
| FR-02-5 | 期限切れ・期限間近の在庫品を強調表示し、該当カテゴリを自動展開する |
| FR-02-6 | 各品目から登録画面へ遷移して在庫品を追加できる |
| FR-02-7 | キーワード検索・不足品目フィルタでリストを絞り込める |
| FR-02-8 | スワイプ操作で品目を非表示にできる（設定から再表示可能） |
| FR-02-9 | スターターセット（8品目）と本格備蓄（全品目）のモードを切り替えられる |
| FR-02-10 | スターターモードで達成率 80% 以上になると本格備蓄への移行バナーを表示する |

### 2.4 備蓄品の登録・編集（FR-03）

| ID | 要件 |
|---|---|
| FR-03-1 | 商品名・数量・単位・消費期限を入力して登録できる |
| FR-03-2 | マスター品目から追加する場合、品目名・単位はプリセットされ編集不可 |
| FR-03-3 | 既存の備蓄品を編集・削除できる |
| FR-03-4 | 全角数字は自動で半角に変換する。数値以外の入力はエラー表示する |
| FR-03-5 | 必須項目（品目名・商品名・数量・単位）が未入力の場合はエラーを表示する |

### 2.5 ホーム画面（FR-04）

| ID | 要件 |
|---|---|
| FR-04-1 | 全体の備蓄達成率（%・品目数）をプログレスバーで表示する |
| FR-04-2 | 期限切れ・期限間近の在庫品をアラートカードで最大3件表示する |
| FR-04-3 | くらし方が未設定の場合は設定を促すメッセージを表示する |
| FR-04-4 | 各主要ページへのグリッドメニューを表示する |
| FR-04-5 | 優先アクション（補充上位3件・期限間近上位2件）をカードで表示する |
| FR-04-6 | 備蓄期間モードをホーム画面から切り替えられる |
| FR-04-7 | 棚卸しリマインダーバナーを表示する（前日・当日・超過時） |
| FR-04-8 | 次回棚卸し予定日を常設表示する（バナー非表示時） |

### 2.6 備蓄クエスト（FR-05）

| ID | 要件 |
|---|---|
| FR-05-1 | 優先アクション・不足品目・期限間近品目を3セクションで表示する |
| FR-05-2 | 不足品目は不足率の高い順に表示する |
| FR-05-3 | 「追加する」タップで登録画面へ遷移し、品目情報をプリセットする |
| FR-05-4 | 「使った」タップで数量入力モーダルを表示し、ローリングストックを記録する |
| FR-05-5 | 使用量が在庫量を超える場合はリストから品目を削除する |

### 2.7 設定（FR-06）

| ID | 要件 |
|---|---|
| FR-06-1 | 期限アラートを表示する日数を備蓄モード別に設定できる |
| FR-06-2 | 棚卸し予定日・棚卸し間隔（日）を設定できる |
| FR-06-3 | 備蓄モード（スターター / 本格備蓄）を切り替えられる |
| FR-06-4 | 非表示にした品目を一覧表示し、再表示できる |
| FR-06-5 | データをJSONファイルとしてエクスポート・インポートできる |
| FR-06-6 | 全データをリセット（削除）できる（確認ダイアログ付き） |
| FR-06-7 | カスタム品目リストの編集画面へ遷移できる |

### 2.8 カスタム品目管理（FR-07）

| ID | 要件 |
|---|---|
| FR-07-1 | デフォルトの推奨品目リストを閲覧できる |
| FR-07-2 | 新しいカスタム品目（品目名・カテゴリ・単位・1日あたりの目標量）を追加できる |
| FR-07-3 | 追加したカスタム品目を編集・削除できる（関連在庫も同時削除・確認ダイアログ付き） |

## 3. 非機能要件

| 区分 | 要件 |
|---|---|
| **プライバシー** | データを外部送信しない。`localStorage` のみ使用 |
| **オフライン** | PWA（Service Worker）によりネットワーク不要で動作する |
| **モバイル対応** | タップ操作を主眼に置いたUI。ボタンのタップ領域は44px以上を確保する |
| **パフォーマンス** | ビルド不要の静的ファイル構成（HTML + CSS + ESモジュール） |
| **拡張性** | データアクセスを `storage.js` 一点に集約。将来のクラウド移行に対応できる構造 |
| **アクセシビリティ** | 確認ダイアログに `role="dialog"` `aria-modal="true"` を付与する |

## 4. 備蓄品マスターリスト要件

### 4.1 計算基準

内閣府・東京都防災指針に基づく推奨量を採用する。

| 品目 | 根拠 |
|---|---|
| 水 | 1人1日3L（飲料水・調理用）|
| 食事 | 1人1日3食 |
| 携帯トイレ | 1人1日5回 |
| カセットボンベ | 1本で約60分使用。1日80分調理で1.34本/日 |

### 4.2 対象者フィルタリング

家族構成に含まれない対象者の品目は表示しない（例：乳幼児がいなければ粉ミルクを非表示）。

### 4.3 対象者区分

- **全員**: `totalPeople` でカウント
- **成人・子ども**: `adults + children`
- **乳幼児**: `infants`（0〜2歳）
- **高齢者**: `elderly`（65歳以上）
- **女性（月経対象年齢）**: `females_menstrual`（12歳以上65歳未満の女性）
- **ペット**: `pets`（頭数合計）

### 4.4 スターターセット（8品目）

`STARTER_IDS` として `masterData.js` に定義。初心者向けの最低限品目。

| ID | 品目名 |
|---|---|
| `water` | 水 |
| `staple_food` | レトルトご飯・乾麺など |
| `side_dish` | 缶詰・レトルト食品 |
| `portable_toilet` | 携帯トイレ・簡易トイレ |
| `flashlight` | 懐中電灯・ランタン |
| `batteries` | 乾電池 |
| `mobile_battery` | 携帯充電器 |
| `first_aid_kit` | 救急箱 |
