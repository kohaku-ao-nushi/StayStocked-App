# 要件定義書 — StayStocked

## 1. プロダクト概要

**StayStocked** は家庭の防災備蓄を管理するモバイルファーストの Web アプリケーション。
家族構成・ペット情報をもとに備蓄品の推奨量を自動計算し、現在の在庫との過不足をひと目で把握できる。

- **対象ユーザー**: 防災備蓄に取り組む一般家庭
- **動作環境**: スマートフォンのモバイルブラウザ（PWA 対応予定）
- **データ保管**: 全データをブラウザの `localStorage` に保存。外部サーバーには一切送信しない

## 2. 機能要件

### 2.1 くらし方設定（FR-01）

| ID | 要件 |
|---|---|
| FR-01-1 | 家族人数（1〜9人）を選択できる |
| FR-01-2 | 人数分の家族構成カードを動的に表示し、性別・年代を入力できる |
| FR-01-3 | 年代区分は「乳幼児（0〜2歳）」「子ども（3〜17歳）」「成人（18〜64歳）」「高齢者（65歳以上）」の4区分 |
| FR-01-4 | ペットの頭数（0〜9頭）を設定できる |
| FR-01-5 | 保存時に `localStorage` へ書き込み、ホーム画面へ遷移する |
| FR-01-6 | 保存済みデータがある場合は設定値をプリセット表示する |

### 2.2 備蓄リスト（FR-02）

| ID | 要件 |
|---|---|
| FR-02-1 | 備蓄期間モード（3日分 / 1週間 / 2週間）を切り替えられる |
| FR-02-2 | 家族構成に応じた必要量を品目ごとに自動計算して表示する |
| FR-02-3 | 現在の在庫量と必要量を比較し、プログレスバーで達成率を表示する |
| FR-02-4 | 品目をカテゴリ別にグルーピングして表示する |
| FR-02-5 | 期限切れ・期限間近の在庫品を強調表示する |
| FR-02-6 | 各品目から登録画面へ遷移して在庫品を追加できる |
| FR-02-7 | マスターリストにない品目も個別追加できる |

### 2.3 備蓄品の登録・編集（FR-03）

| ID | 要件 |
|---|---|
| FR-03-1 | 商品名・数量・単位・消費期限を入力して登録できる |
| FR-03-2 | マスター品目から追加する場合、品目名・単位はプリセットされ編集不可 |
| FR-03-3 | 既存の備蓄品を編集・削除できる |
| FR-03-4 | マスターにない品目は品目名・カテゴリも手動入力できる |
| FR-03-5 | 必須項目（品目名・商品名・数量・単位）が未入力の場合はエラーを表示する |

### 2.4 ホーム画面（FR-04）

| ID | 要件 |
|---|---|
| FR-04-1 | 全体の備蓄達成率（%・品目数）をプログレスバーで表示する |
| FR-04-2 | 期限切れ・期限間近の在庫品をアラートカードで最大3件表示する |
| FR-04-3 | くらし方が未設定の場合は設定を促すメッセージを表示する |
| FR-04-4 | 各主要ページへのグリッドメニューを表示する |

### 2.5 設定（FR-05）

| ID | 要件 |
|---|---|
| FR-05-1 | 期限アラートを表示する日数を備蓄モード別に設定できる（デフォルト：3日分→7日前、1週間→14日前、2週間→30日前） |
| FR-05-2 | 全データをリセット（削除）できる（確認ダイアログ付き） |
| FR-05-3 | カスタム品目リストの編集画面へ遷移できる |

### 2.6 カスタム品目管理（FR-06）

| ID | 要件 |
|---|---|
| FR-06-1 | デフォルトの推奨品目リストを閲覧できる |
| FR-06-2 | 新しいカスタム品目（品目名・カテゴリ・単位・1人1日あたりの目標量）を追加できる |
| FR-06-3 | 追加したカスタム品目を削除できる（関連在庫も同時削除・確認ダイアログ付き） |

---

## 3. 非機能要件

| 区分 | 要件 |
|---|---|
| **プライバシー** | データを外部送信しない。`localStorage` のみ使用 |
| **オフライン** | ネットワーク不要で動作する（静的ファイルのみ） |
| **モバイル対応** | タップ操作を主眼に置いたUI。ボタンのタップ領域は十分な大きさを確保する |
| **パフォーマンス** | ビルド不要の静的ファイル構成（HTML + CSS + ESモジュール） |
| **拡張性** | 将来の PWA 化・IndexedDB 移行に備え、データアクセスを `storage.js` 一点に集約する |

## 4. 備蓄品マスターリスト要件

### 4.1 計算基準

内閣府・東京都防災指針に基づく推奨量を採用する。

| 品目 | 根拠 |
|---|---|
| 水 | 1人1日3L（飲料水・調理用）|
| 食事 | 1人1日3食 |
| 携帯トイレ | 1人1日5回 |
| カセットボンベ | 1本で約60分使用。1日2〜3回の調理で1.3本/日 |

### 4.2 対象者フィルタリング

家族構成に含まれない対象者の品目は表示しない（例：乳幼児がいなければ粉ミルクを非表示）。

### 4.3 対象者区分

- **全員**: totalPeople でカウント
- **成人・子ども**: adults + children
- **乳幼児**: infants（0〜2歳）
- **高齢者**: elderly（65歳以上）
- **女性（月経対象年齢）**: females_menstrual（12歳以上65歳未満の女性）
- **ペット**: pets（頭数）

### 4.4 必須カバレッジ品目

以下は必須掲載品目として定義する（現バージョンで不足しているものを含む）。

**食品・飲料**
- 水、主食（レトルト等）、副食（缶詰等）、粉ミルク・液体ミルク、おかゆ・介護食、お菓子・嗜好品

**衛生用品**
- 携帯トイレ、ティッシュ、トイレットペーパー、除菌ウェットティッシュ、ウェットボディタオル、マスク、おむつ、生理用品、アルコール消毒液、ビニール手袋

**医療・医薬**
- 救急箱、常備薬・持病の薬、体温計

**生活用品**
- カセットコンロ、カセットボンベ、食品用ラップ、ポリ袋、懐中電灯・ランタン、ヘッドライト、乾電池、携帯充電器、防災ラジオ、ガムテープ、軍手

**防災装備**
- 防災頭巾・ヘルメット、ホイッスル

**書類・現金**
- 重要書類のコピー（保険証・通帳等）、現金・小銭

**ペット用品**
- ペットフード、ペット用トイレ用品
