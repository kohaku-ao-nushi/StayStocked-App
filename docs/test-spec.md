# テスト仕様書 — クラソナ（Crasona）

> **最終更新**: 2026-05-31
> **現状**: テストフレームワーク未導入。本書は手動テスト手順と、将来的なユニットテスト導入時の仕様を定義する。

---

## 1. テスト方針

| 種別 | 対象 | 方法 |
|---|---|---|
| **ユニットテスト** | `buildCalcParams`, `calc` 関数 | Vitest（将来導入） |
| **統合テスト** | ページ描画・データ保存の一連フロー | 手動テスト |
| **回帰テスト** | バグ修正後の動作確認 | 手動チェックリスト |

## 2. 手動テスト — 機能テスト

### 2.1 初回起動・オンボーディング

| # | 操作 | 期待される動作 |
|---|---|---|
| 1 | アプリを初めて開く（localStorage 空） | オンボーディング画面が表示される |
| 2 | 「次へ」を3回タップする | ステップが進み、最後に「はじめる →」ボタンが表示される |
| 3 | 「はじめる →」をタップする | くらし方設定画面へ遷移する |
| 4 | 2回目以降の起動 | オンボーディングはスキップされホーム画面が表示される |
| 5 | 「スキップ」をタップする | くらし方設定画面へ遷移する |

### 2.2 くらし方設定

| # | 操作 | 期待される動作 |
|---|---|---|
| 1 | 人数を −/＋ ボタンで変更する | 人数分のプロフィールカードが動的に増減する |
| 2 | 各プロフィールの性別・年代を選択して保存する | 「くらし方を保存しました」トーストが表示されホームへ遷移 |
| 3 | ペットのエントリを追加する | 種類・単位・頭数フォームが追加される |
| 4 | ホームに戻る | 備蓄達成率が表示される |

### 2.3 備蓄リスト

| # | 操作 | 期待される動作 |
|---|---|---|
| 1 | 備蓄リストを開く | カテゴリ別に品目が表示され全体達成率が表示される |
| 2 | モードを「1週間」に切り替える | 必要量が7日分になり達成率が再計算される |
| 3 | キーワード検索に入力する | 一致する品目のみ表示され、フォーカスが外れない |
| 4 | 「不足のみ」フィルタを ON にする | 在庫が不足している品目のみ表示される |
| 5 | 品目を左スワイプする | 「非表示」ボタンが現れる |
| 6 | 「非表示」をタップする | 品目がリストから消える |
| 7 | くらし方に乳幼児がいない場合 | 「粉ミルク」「おむつ」が表示されない |
| 8 | くらし方に女性がいない場合 | 「生理用品」が表示されない |
| 9 | 期限切れ品目がある場合 | 赤い強調表示と該当カテゴリの自動展開 |

### 2.4 備蓄品の登録

| # | 操作 | 期待される動作 |
|---|---|---|
| 1 | 品目の「＋ この品目を追加」をタップ | 品目名・単位がプリセットされた登録フォームが開く |
| 2 | 商品名・数量を入力して「追加する」をタップ | 登録され備蓄リストに戻る |
| 3 | 登録後に備蓄リストを確認する | 登録した数量が反映されプログレスバーが増加する |
| 4 | 数量フィールドに全角数字を入力する | 半角に自動変換される |
| 5 | 数量フィールドに文字を入力する | `.input-error` が付与されエラー表示になる |
| 6 | 必須項目を空白にして「追加する」をタップ | エラートーストが表示される |
| 7 | 登録済みの在庫品をタップ | 編集フォームが開き既存値がプリセットされている |
| 8 | 編集画面で「削除する」をタップ | 確認ダイアログ後に削除されリストに戻る |

### 2.5 びちくえ！（備蓄クエスト）

| # | 操作 | 期待される動作 |
|---|---|---|
| 1 | びちくえ！を開く | 優先アクション・不足品目・期限間近の3セクションが表示される |
| 2 | 不足品目の「追加する」をタップ | 登録画面へ遷移し品目がプリセットされている |
| 3 | 期限品目の「使った」をタップ | 数量入力モーダルが表示される |
| 4 | 数量を入力して「記録する」をタップ | 在庫が減算されリストが更新される |
| 5 | 在庫量より多い数量を入力して記録する | 品目がリストから削除され「使い切りました」トーストが表示される |

### 2.6 ホーム画面

| # | 操作 | 期待される動作 |
|---|---|---|
| 1 | くらし方未設定でホームを開く | 設定を促すメッセージとリンクが表示される |
| 2 | 備蓄期間モードを切り替える | 達成率が即時更新される |
| 3 | 棚卸し日を過去の日付に設定してホームを開く | リマインダーバナーが表示される |
| 4 | 「確認した」をタップする | バナーが消え次回棚卸し日が自動更新される |
| 5 | スターターモードで80%以上達成する | 「本格備蓄に移行しますか？」バナーが表示される（備蓄リスト） |

### 2.7 設定

| # | 操作 | 期待される動作 |
|---|---|---|
| 1 | 期限アラート日数を変更して保存する | 「設定を保存しました」トーストが表示される |
| 2 | 棚卸し予定日・間隔を設定して保存する | 「リマインダーを保存しました」トーストが表示される |
| 3 | 備蓄モードを「本格備蓄」に切り替える | 確認ダイアログ後に切り替わりバッジが変わる |
| 4 | 非表示品目の「再表示」をタップする | 品目が備蓄リストに戻る |
| 5 | エクスポートをタップする | `crasona_backup_YYYYMMDD.json` がダウンロードされる |
| 6 | 正しい形式のJSONをインポートする | 確認ダイアログ後にデータが反映されホームへ遷移する |
| 7 | 不正なJSONをインポートする | エラートーストが表示される |
| 8 | 全データをリセットする | 確認ダイアログ後にリセットされ初回起動状態に戻る |

### 2.8 カスタム品目管理

| # | 操作 | 期待される動作 |
|---|---|---|
| 1 | 設定 → 品目リストを編集する | デフォルト品目の一覧とカスタム品目の追加フォームが表示される |
| 2 | 新しい品目（名前・カテゴリ・単位）を入力して追加する | 「追加しました」トーストが表示されカスタムリストに追加される |
| 3 | 追加したカスタム品目が備蓄リストに表示されることを確認する | 備蓄リストの該当カテゴリに表示される |
| 4 | カスタム品目を削除する | 確認ダイアログ後に削除される。関連する在庫品も消える |

## 3. 手動テスト — エッジケース

| # | ケース | 期待される動作 |
|---|---|---|
| 1 | 備蓄期間モードを変更してページを離れ再度訪問する | 変更後のモードが維持されている |
| 2 | 消費期限に今日の日付を入力する | 「期限切れ」として扱われる |
| 3 | 同じ品目を複数回登録する | 備蓄リストの現在量に合算される |
| 4 | localStorage を手動で削除してリロードする | 初回起動状態になりエラーが発生しない |
| 5 | localStorage に不正な JSON が格納されている場合 | デフォルト値で起動しエラーが発生しない |
| 6 | 存在しない URL ハッシュ（例：`#unknown`）にアクセスする | ホームにリダイレクトされる |
| 7 | 備蓄リストで複数のカテゴリを同時に開いた状態でフィルタを操作する | フィルタ結果に応じて展開状態が保たれる |

## 4. 手動テスト — UI・アクセシビリティ

| # | チェック項目 |
|---|---|
| 1 | モバイル幅（375px）でレイアウトが崩れないこと |

| 6 | ページ遷移時にスクロール位置が先頭に戻ること |
| 7 | ボトムナビの現在ページに is-active が付くこと |

## 5. 回帰テストチェックリスト（マスターデータ変更後）

マスターデータ（`masterData.js`）を変更したときは必ず以下を確認すること。

- [ ] くらし方設定: 成人男性1人 + 成人女性1人 + 子ども1人 + 乳幼児1人 + ペット1頭 を設定する
- [ ] 備蓄リスト（3日分）: 全カテゴリの全品目が表示されることを確認する
- [ ] 備蓄リスト（3日分）: 各品目の必要量が設計書の計算式と一致することを確認する
- [ ] くらし方設定: 乳幼児なし・女性なし・ペットなし の構成に変更する
- [ ] 備蓄リスト: 粉ミルク・おむつ・生理用品・ペット用品が非表示になることを確認する
- [ ] 備蓄リスト（1週間）・（2週間）に切り替えて必要量が正しくスケールされることを確認する
- [ ] びちくえ！: 不足品目が正しく表示されることを確認する
