| days | females_menstrual | 期待値 | 根拠 |

## 3. 手動テスト — 機能テスト

### 3.1 初回起動・セットアップ

| # | 操作 | 期待される動作 |
|---|---|---|
| 1 | アプリを初めて開く | ホーム画面に「くらし方を設定しましょう」メッセージが表示される |
| 2 | くらし方ページで人数を1〜9人に変更する | 人数分のプロフィールカードが動的に増減する |
| 3 | 各プロフィールの性別・年代を選択して保存する | トースト通知「くらし方を保存しました」が表示され、ホームへ遷移する |
| 4 | ホームに戻る | 備蓄達成率が表示される（初期は 0%） |

### 3.2 備蓄リスト

| # | 操作 | 期待される動作 |
|---|---|---|
| 1 | 備蓄リストを開く | カテゴリ別に品目が表示され、全体達成率が表示される |
| 2 | モードを「1週間」に切り替える | 必要量が7日分に変わり、達成率が再計算される |
| 3 | くらし方に乳幼児がいない場合 | 「粉ミルク・液体ミルク」「おむつ」が表示されない |
| 4 | くらし方に女性（成人）がいない場合 | 「生理用品」が表示されない |
| 5 | 期限切れ品目がある場合 | 赤い強調表示と `<details>` が自動展開される |

### 3.3 備蓄品の登録

| # | 操作 | 期待される動作 |
|---|---|---|
| 1 | 「＋ この品目を追加」をタップする | 品目名・単位がプリセットされた登録フォームが開く |
| 2 | 商品名・数量を入力して「追加する」をタップ | 登録され、備蓄リストに戻る |
| 3 | 登録後に備蓄リストを確認する | 登録した数量が現在量に反映されプログレスバーが増加する |
| 4 | 必須項目を空白にして「追加する」をタップ | エラートースト「品目名・商品名・数量・単位は必須です」が表示される |
| 5 | 登録済みの在庫品をタップする | 編集フォームが開き、既存値がプリセットされている |
| 6 | 編集画面で「削除する」をタップする | 確認ダイアログが表示され、OKで削除されリストに戻る |

### 3.4 カスタム品目

| # | 操作 | 期待される動作 |
|---|---|---|
| 1 | 設定 → 品目リストを編集する | デフォルト品目の一覧とカスタム品目の追加フォームが表示される |
| 2 | 新しい品目（名前・カテゴリ・単位）を入力して追加する | 「追加しました」トーストが表示され、カスタムリストに追加される |
| 3 | 追加したカスタム品目が備蓄リストに表示されることを確認する | 備蓄リストの該当カテゴリに表示される |
| 4 | カスタム品目を削除する | 確認ダイアログ後に削除される。関連する在庫品も消える |

### 3.5 設定・リセット

| # | 操作 | 期待される動作 |
|---|---|---|
| 1 | 期限アラート日数を変更して保存する | トーストが表示され、次回から新しい日数が適用される |
| 2 | 全データをリセットする | 確認ダイアログ後にリセットされ、初回起動状態に戻る |
| 3 | リセット後にホームを確認する | 「くらし方を設定しましょう」メッセージが表示される |

## 4. 手動テスト — エッジケース

| # | ケース | 期待される動作 |
|---|---|---|
| 1 | 備蓄期間モードを変更してページを離れ、再度訪問する | 変更後のモードが維持されている |
| 2 | 消費期限に今日の日付を入力する | 「期限切れ」として扱われる |
| 3 | 数量に 0 を入力して登録する | 登録でき、達成率に 0 として反映される |
| 4 | 同じ品目を複数回登録する | 備蓄リストの現在量に合算される |
| 5 | localStorage を手動で削除してリロードする | 初回起動状態になりエラーが発生しない |
| 6 | localStorage に不正な JSON が格納されている場合 | デフォルト値で起動しエラーが発生しない |
| 7 | 存在しない URL ハッシュ（例：`#unknown`）にアクセスする | ホームにリダイレクトされる |

## 6. 回帰テストチェックリスト（マスターデータ変更後）

マスターデータ（`masterData.js`）を変更したときは必ず以下を確認すること。

- [ ] くらし方設定: 成人男性1人 + 成人女性1人 + 子ども1人 + 乳幼児1人 + ペット1頭 を設定する
- [ ] 備蓄リスト（3日分）: 全カテゴリの全品目が表示されることを確認する
- [ ] 備蓄リスト（3日分）: 各品目の必要量が設計書の計算式と一致することを確認する
- [ ] くらし方設定: 乳幼児なし・女性なし・ペットなし の構成に変更する
- [ ] 備蓄リスト: 粉ミルク・おむつ・生理用品・ペット用品が非表示になることを確認する
- [ ] 備蓄リスト（1週間）・（2週間）に切り替えて必要量が正しくスケールされることを確認する
